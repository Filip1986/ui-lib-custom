Toast

The Toast component provides a lightweight notification overlay driven by an injectable ToastService. Place one <ui-lib-toast /> in your app shell and call toastService.add() from anywhere in the application.

Severities

Four severity levels map to distinct colour palettes and icons.

Multiple Toasts

Multiple toasts can be displayed simultaneously and stack in the container.

Sticky & Non-Closable

Set sticky: true to prevent auto-dismissal. Set closable: false to hide the close button.

Auto-dismiss Duration

Control how long a toast stays visible with the life property (milliseconds). Per-message life overrides the container default.

Position & Variant

Choose a position and design variant, then trigger a toast to see it in action.

Position
@for (pos of positions; track pos) { }
Variant
@for (v of variants; track v) { }

Keyed Containers

Use the key input on both the container and the message to route toasts to a specific container. Useful for displaying toasts in different screen regions simultaneously.

Toast API

Toast container inputs

Input Type Default Description
position ToastPosition 'top-right' Screen position of the toast container.
life number 3000 Default auto-dismiss duration (ms) for all messages.
variant ToastVariant | null null Design variant. Falls back to ThemeConfigService global variant.
key string | null null Only renders messages whose key matches this value.
styleClass string | null null Extra CSS class(es) applied to the host element.

ToastMessage interface

Property Type Default Description
id string auto Unique identifier — auto-generated if omitted.
key string — Routes the message to the matching container key.
severity ToastSeverity 'info' Controls colour palette and default icon.
summary string — Bold headline of the notification.
detail string — Body text of the notification.
life number container life Per-message auto-dismiss duration (ms).
sticky boolean false When true, the toast never auto-dismisses.
closable boolean true When false, the close button is hidden.
icon string severity default Custom icon name overriding the severity default.
styleClass string — Extra CSS class(es) on the item element.

ToastService methods

Method Signature Description
add (message: ToastMessage) => void Enqueue a new toast notification.
remove (id: string) => void Remove a toast by its ID.
clear (key?: string) => void Clear all messages, or only those matching a key.
