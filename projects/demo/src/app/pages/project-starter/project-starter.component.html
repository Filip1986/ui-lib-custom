Three ways to consume ui-lib-custom locally before publishing to a registry.

Step 0 — build the library

Every method below requires a fresh build output in dist/ui-lib-custom/.

Method 1 — npm link recommended for active dev

Creates a global symlink so your consumer project always sees the latest build. Re-run npm run build in this workspace and the consumer picks up changes immediately.

Method 2 — local file path

Point your package.json directly at the dist/ folder. No global symlink needed — good for one-off integration tests.

Method 3 — publish to npm

When you are ready to share the library, update the package name and publish from dist/ui-lib-custom/.
