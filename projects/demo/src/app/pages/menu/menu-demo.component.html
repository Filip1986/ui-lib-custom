Menu

A flexible navigation panel supporting both inline and popup modes. Items can be organised into labelled groups, separated by dividers, and decorated with icons. Implements the roving tabindex pattern for accessible keyboard navigation.

Overview

Menu renders an accessible, keyboard-navigable menu panel that supports flat and grouped item lists. Two modes are available:

  • Inline mode (default) — the panel is always visible in the document flow.
  • Popup mode ([popup]="true") — the panel floats as a fixed overlay anchored to the trigger element; control visibility with toggle(), show(), or hide().

Unlike TieredMenu, the Menu component supports labelled groups via top-level items with an items array. These render as section headers within a flat panel rather than flyout sub-menus.

With Separator

Add { separator: true } to insert a horizontal divider between groups of related items. Separators use role="separator" for screen reader support.

Disabled Items

Set disabled: true on any item to make it non-interactive and visually muted. Disabled items receive aria-disabled="true" and are excluded from the roving tabindex navigation.

Grouped Items

Provide an items array on a top-level entry to create a labelled section. The parent's label becomes a visual group header; the group receives role="group" with an aria-label matching the header text.

Popup Mode

Set [popup]="true" and control visibility via toggle(event), show(event), or hide(). Wire the trigger button with aria-haspopup, aria-expanded, and aria-controls for screen reader support.

Methods

NameParametersDescription
toggle(event) MouseEvent Toggles the popup open or closed anchored to the trigger. No-op in inline mode.
show(event) MouseEvent Shows the popup anchored to event.currentTarget. No-op in inline mode.
hide() — Hides the popup and restores focus to the trigger element. No-op in inline mode.

MenuItem

PropertyTypeDescription
label string? Display text for the item or group header.
icon string? Icon class rendered in a decorative <span aria-hidden="true">.
disabled boolean? When true, item is non-interactive; gets aria-disabled="true" and excluded from keyboard navigation.
separator boolean? Renders a <li role="separator"> horizontal divider.
visible boolean? When explicitly false, excludes the item from rendering.
items MenuItem[]? Child items — makes the parent a labelled group header (role="group").
url string? Renders the item as <a href="...">.
target string? target attribute for URL-based items (e.g. '_blank').
styleClass string? Extra CSS class added to the rendered link element.
command function? Callback invoked when the item is activated (click or keyboard).

Keyboard Navigation

KeyAction
ArrowDown / ArrowUpMove focus to the next or previous enabled item (wraps, skips disabled).
Home / EndJump to the first or last enabled item.
Enter / SpaceActivate the focused item.
EscapeClose the popup panel and restore focus to the trigger element. No-op in inline mode.
TabClose the popup and move focus naturally to the next focusable element.

Notes

  • The roving tabindex pattern is used — only one item has tabindex="0" at a time, all others use tabindex="-1". This reserves Tab for leaving the menu rather than stepping through every item.
  • Focus is restored to the element that triggered the popup when the panel closes via Escape, Tab, or a click outside.
  • Popup trigger ARIA wiring is the consumer's responsibility — expose menuId and isVisible() for binding aria-controls and aria-expanded on the trigger.
  • The popup slide-in animation is disabled when prefers-reduced-motion: reduce is active.
