RadioButton

Accessible radio button that implements ControlValueAccessor — works with ngModel and reactive forms. Supports arrow-key group navigation, three design variants, three sizes, and a filled appearance.

Basic

All buttons in a group share the same name. Bind the shared group value with [(ngModel)]. Wrap the group in a <fieldset> / <legend> for accessible group labeling.

City
@for (city of cities; track city.value) { }

Selected: {{ selectedCity || 'none' }} @if (lastChangeEvent) {  — last change: {{ lastChangeEvent.value }} }

Variants

Three design variants: material, bootstrap, and minimal. When variant is null (default), it inherits from ThemeConfigService.

Sizes

Three sizes: sm, md (default), and lg.

Filled Appearance

Set appearance="filled" to use a solid checked background instead of the default outlined ring style.

@for (option of [{label: 'One', value: 'one'}, {label: 'Two', value: 'two'}, {label: 'Three', value: 'three'}]; track option.value) { }

Disabled

Set [disabled]="true" on individual buttons. Disabled radios are automatically skipped during arrow-key navigation.

With ngModel

All buttons in the group share the same [(ngModel)] binding. The button whose [value] matches the bound value becomes checked.

Fruit
@for (fruit of fruits; track fruit.value) { }

Selected fruit: {{ selectedFruit }}

Reactive Forms

Use formControlName or [formControl] for reactive form integration. FormControl.disable() propagates through setDisabledState.

Plan
@for (plan of plans; track plan.value) { }

Form value: {{ planForm.value | json }}

Inputs

Input Type Default Description
label string | null null Visible label text. Content projection is an alternative.
inputId string | null null Forwarded to the native <input> id.
name string | null null Must be identical across all buttons in a group.
value unknown null The value this radio button represents in the group.
required boolean false Sets aria-required on the native input.
readonly boolean false Prevents value changes while remaining visible.
disabled boolean false Sets aria-disabled on the native input. Skipped during arrow-key navigation.
tabindex number 0 Applied when the radio is checked (roving tabindex pattern).
autofocus boolean false Focuses this radio after first render.
ariaLabel string | null null Used when no visible label is provided.
ariaLabelledby string | null null Explicit override; takes precedence over the auto-generated label id.
variant 'material' | 'bootstrap' | 'minimal' | null null Design variant. Inherits from ThemeConfigService when null.
size 'sm' | 'md' | 'lg' 'md' Component density.
appearance 'outlined' | 'filled' 'outlined' Visual style — filled uses a solid checked background.

Outputs

Output Payload Description
(change) RadioButtonChangeEvent Fires when this button is selected. Payload: { value, originalEvent }.
(focus) FocusEvent Emitted when the radio receives focus.
(blur) FocusEvent Emitted when the radio loses focus.

ARIA Attributes

Element Attribute Value / Notes
Native <input type="radio"> role="radio" Implicit via input type — native browser semantics.
Native input aria-checked Managed natively by the browser based on checked property.
Native input aria-required Set when [required]="true".
Native input aria-disabled Set when [disabled]="true".
Native input id From inputId input; auto-generated when not provided.
<label> for Points to the native input id — links label and control.
Native input tabindex Checked radio gets tabindex="0"; unchecked radios get tabindex="-1" (roving tabindex).

Group labeling is consumer responsibility. Wrap radio buttons in a <fieldset> / <legend> (recommended) or a role="radiogroup" element with aria-labelledby.

Keyboard Interaction

Key Action
Tab Enters the group; browser focuses the checked radio (or the first if none checked).
Shift+Tab Leaves the group.
ArrowDown / ArrowRight Moves focus to the next non-disabled radio, wrapping around; selects it.
ArrowUp / ArrowLeft Moves focus to the previous non-disabled radio, wrapping around; selects it.
Space Selects the focused radio (native browser behaviour).

Notes

  • Disabled radios are automatically skipped during arrow-key navigation.
  • Group labeling is consumer responsibility — use <fieldset> / <legend> or role="radiogroup" with aria-labelledby to give the group an accessible name.
  • When no visible label is supplied, use ariaLabel or ariaLabelledby to provide an accessible name for the individual button.
  • All transitions respect prefers-reduced-motion: reduce.
