TieredMenu

A hierarchical flyout menu supporting arbitrarily deep nesting. Works in both inline and popup modes. Fully accessible with roving focus, ARIA roles, and keyboard navigation.

Overview

TieredMenu renders an accessible, keyboard-navigable menu panel that supports arbitrarily deep nesting. Items with an items array open a flyout sub-panel on hover or ArrowRight. Two modes are available:

  • Inline mode (default) — the panel renders in the document flow; ideal for sidebars or always-visible navigation.
  • Popup mode ([popup]="true") — the panel floats as a fixed overlay anchored to the trigger element; control visibility with toggle(), show(), or hide().

Wiring ARIA attributes on the popup trigger is the consumer's responsibility — see the Popup Mode section for the required pattern.

Variants

Choose from material, bootstrap, or minimal. When variant is null (the default), the value falls back to ThemeConfigService.

material bootstrap minimal

Sizes

Three sizes are available: sm, md (default), and lg. The size token affects font size, padding, and minimum panel width.

sm md lg

Item States

Items support disabled, visible: false (excluded from render), separators, and command callbacks. Disabled items receive aria-disabled="true" and tabindex="-1".

API

Inputs

popup boolean false When true, renders as a floating fixed overlay. Control visibility with toggle(), show(), or hide().

Outputs

NamePayloadDescription
itemClick TieredMenuItemCommandEvent Emitted when a non-disabled leaf item is activated (click or keyboard).
menuShow MouseEvent | KeyboardEvent Emitted when the popup panel becomes visible. Popup mode only.
menuHide void Emitted when the popup panel is hidden. Popup mode only.

Methods

NameParametersDescription
show(event) MouseEvent | KeyboardEvent Opens the popup anchored to event.currentTarget. No-op in inline mode.
hide() — Closes the popup and restores focus to the trigger element. No-op in inline mode.
toggle(event) MouseEvent | KeyboardEvent Calls show() or hide() based on current visibility. No-op in inline mode.

TieredMenuItem

PropertyTypeDescription
label string? Display text for the item.
icon string? Icon class applied to a <span aria-hidden="true">, e.g. 'pi pi-file'.
items TieredMenuItem[]? Child items — renders a right-arrow caret and opens a flyout sub-panel on hover or ArrowRight.
disabled boolean? When true, item is non-interactive; gets aria-disabled="true" and tabindex="-1".
separator boolean? Renders a <li role="separator"> horizontal divider instead of a link.
visible boolean? When explicitly false, excludes the item from the rendered list.
url string? Renders the item as <a href="...">.
target string? target attribute for URL-based items (e.g. '_blank').
command function? Callback invoked when the item is activated.
styleClass string? Extra CSS class added to the <li> element.

Accessibility

ARIA Attributes

ElementAttributeValueNotes
Root <ul> role "menu" Identifies the list as a menu widget.
Root <ul> aria-label ariaLabel input Accessible name for the root panel, defaults to "Menu".
Nested <ul> aria-label Parent item label Each sub-panel is named after its parent item for screen reader context.
<li> wrapper role "none" Removes implicit list-item semantics, preserving the menuitem role on the link.
Leaf <a> role "menuitem" Identifies each interactive item as a menu item.
Parent <a> aria-haspopup "menu" Signals that the item controls a submenu.
Parent <a> aria-expanded "true" | "false" Reflects whether the flyout sub-panel is currently open.
Disabled <a> aria-disabled "true" Communicates non-interactive state without removing focus management.
Separator <li> role "separator" Conveys structural grouping without aria-hidden.

Keyboard Navigation

KeyAction
ArrowDown / ArrowUpNavigate between items in the current list (wraps).
Home / EndJump to the first or last item in the current list.
ArrowRightOpen the flyout for an item with nested children; focus first child item.
ArrowLeftClose the current flyout and return focus to the parent item. No-op at root level.
Enter / SpaceActivate the focused leaf item or toggle a parent's flyout.
EscapeClose the innermost open flyout; propagates up through nested levels. In popup mode, closes the entire panel and restores trigger focus.
TabCloses popup panel and moves focus naturally to the next focusable element.

Notes

  • Focus is restored to the element that triggered the popup when the panel closes via Escape, Tab, or a click outside.
  • Popup trigger ARIA wiring is the consumer's responsibility — the component exposes menuId and isVisible() for binding aria-controls and aria-expanded.
  • Both SCSS files include @media (prefers-reduced-motion: reduce) blocks that disable flyout transition animations.
