ConfirmDialog

A modal confirmation overlay with configurable accept/reject actions. Supports both declarative binding and programmatic use via ConfirmationService.

Last action: {{ lastResult() }}

Basic

Declarative usage — bind [(visible)] and listen to (accepted) / (rejected) outputs.

Dismissable Backdrop

Set dismissableMask to close the dialog by clicking the backdrop.

Non-Closable

Set closable="false" to require the user to choose an action.

With Icon

Pass an icon CSS class via the icon input to render a visual cue beside the message.

Danger Severity

Use acceptSeverity="danger" to signal a destructive action.

Top Position

Use position="top" to anchor the dialog to the top of the viewport.

Programmatic — ConfirmationService

Use ConfirmationService.confirm({ ... }) to trigger the dialog from any component. Place a single <ui-lib-confirm-dialog /> in your layout shell.

Design Variants

Three variants adapt the visual style to your project theme. Use the variant input or rely on ThemeConfigService for the global default.

API Reference

Inputs

Input Type Default Description
visible boolean (model) false Two-way visibility binding
key string '' Targets service calls to this instance
header string 'Confirmation' Dialog title
message string 'Are you sure…' Confirmation message
icon string | null null CSS class for message icon
acceptLabel string 'Yes' Accept button label
rejectLabel string 'No' Reject button label
acceptSeverity ConfirmDialogButtonSeverity 'primary' Accept button colour scheme
rejectSeverity ConfirmDialogButtonSeverity 'secondary' Reject button colour scheme
closable boolean true Show/hide the × close button
dismissableMask boolean false Close on backdrop click
blockScroll boolean true Lock body scroll while open
position ConfirmDialogPosition 'center' Panel position on the viewport
defaultFocus ConfirmDialogDefaultFocus 'accept' Which button gets initial focus
variant ConfirmDialogVariant | null null Design variant override
styleClass string | null null Extra CSS classes on the host

Outputs

Output Payload Description
accepted void Emitted when the user clicks Accept
rejected void Emitted when the user clicks Reject or closes
