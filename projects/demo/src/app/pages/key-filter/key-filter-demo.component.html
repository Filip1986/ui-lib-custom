KeyFilter

Attribute directive that restricts keyboard input on any native input element to characters matching a built-in preset or a custom regular expression. Supports paste and drag-and-drop filtering, modifier-key pass-through, and a runtime bypass toggle.

a. Alphanumeric

The default preset — allows letters and digits, blocks everything else.

b. Alpha

Letters only — digits and symbols are blocked.

c. Positive Integer

Digits only, no minus sign — use for unsigned whole numbers.

d. Integer

Digits and minus sign — use for signed whole numbers.

e. Number

Digits, minus, and decimal point — use for floating-point values.

f. Hexadecimal

Digits 0–9 and letters a–f (case-insensitive).

g. Money

Digits, minus, decimal point, and thousands separator comma.

h. Email

Characters valid in email addresses per RFC 5321 local-part and domain rules.

i. Custom RegExp

Pass any RegExp to filter per character. This example restricts input to vowels only.

j. Bypass

Set keyFilterBypass to true to suspend all filtering at runtime. Toggle below to see the effect on a pint-filtered input.
