Accessibility

ui-lib-button renders a native <button> element and inherits full keyboard and screen-reader support by default.

  • Tab / Shift + Tab — moves focus to / from the button.
  • Space / Enter — activates the button (native browser behaviour).
  • Disabled buttons ([disabled]="true") are excluded from the tab order automatically.
  • Icon-only buttons must provide ariaLabel — without visible text there is no accessible name for screen readers.
  • ariaPressed — set on toggle buttons to communicate pressed state.
  • ariaChecked — use with role="checkbox" or role="radio".
  • All built-in severity colours are tested to meet WCAG 2.1 AA contrast requirements (4.5 : 1 for text).
  • The ghost, neon, and glass appearances may render below 3 : 1 on some custom backgrounds — test with your palette before shipping.
  • Never rely on colour alone to convey meaning.
  • When [loading]="true", the button is automatically disabled and aria-busy="true" is applied.
  • Pair the loading button with a live region (aria-live="polite") to announce progress to assistive technology.
