Accessibility

ui-lib-button renders a native <button> element and inherits full keyboard and screen-reader support by default.

  • Tab / Shift + Tab — moves focus to / from the button.
  • Space / Enter — activates the button (native browser behaviour).
  • Disabled buttons ([disabled]="true") are excluded from the tab order automatically.
  • Use tabIndex to adjust the order or remove a button from the sequence ([tabIndex]="-1").
  • Icon-only buttons must provide ariaLabel — without visible text there is no accessible name for screen readers.
  • ariaPressed — set to true / false on toggle buttons to communicate pressed state (role="button" + aria-pressed).
  • ariaChecked — use with role="checkbox" or role="radio" when the button represents a checkable item.
  • role — override the implicit ARIA role when using the button element in a specialised context (e.g., role="menuitem").
  • All built-in severity colours are tested to meet WCAG 2.1 AA contrast requirements (4.5 : 1 for text, 3 : 1 for large text / UI components).
  • The ghost appearance may render below 3 : 1 on some custom backgrounds — test with your palette before shipping.
  • Never rely on colour alone to convey meaning: pair colour with an icon and/or visible text.
  • When [loading]="true", the button is automatically disabled so it cannot be re-activated, and the spinner is hidden from screen readers (aria-hidden="true").
  • Pair the loading button with a live region (aria-live="polite") in your template to announce progress or completion to assistive technology.
