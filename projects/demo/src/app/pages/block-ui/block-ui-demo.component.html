Notes

  • Focus prevention: The protected content wrapper receives the inert HTML attribute when blocked = true. This prevents all interactive descendants (buttons, links, inputs) from receiving keyboard focus or pointer events — no custom focus trap is needed.
  • Live region: The mask element carries role="status" and aria-live="polite". Screen readers will announce any text projected via [blockTemplate] when the component becomes blocked (e.g. "Processing…").
  • aria-busy: Signals to assistive technologies that the container is loading or processing, suppressing interaction expectations.
  • All transitions respect prefers-reduced-motion: reduce — the opacity transition is set to 0 duration.
