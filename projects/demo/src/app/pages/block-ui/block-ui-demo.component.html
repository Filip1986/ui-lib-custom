Overlay a mask over any section to prevent user interaction — ideal for loading states, disabled form sections, and confirmation flows. Wrap content in <ui-lib-block-ui> and bind [(blocked)] to toggle the mask. Blocked content receives the inert attribute, making all descendants non-focusable by keyboard.

Custom Mask

Project content into the mask overlay using the [blockTemplate] attribute selector. Ideal for spinners, progress indicators, or status messages. The mask element has role="status" and aria-live="polite", so projected text is announced by screen readers when the component becomes blocked.

Inputs

Input Type Default Description
blocked boolean false Whether the content is blocked. Two-way bindable via [(blocked)]. A model() signal internally.
variant 'material' | 'bootstrap' | 'minimal' | null null Visual variant. Inherits from ThemeConfigService when null.
styleClass string | null null Additional CSS class(es) applied to the host element.
baseZIndex number 0 Base z-index for the mask layer. Uses the CSS variable default when 0.

Content Slots

Slot Description
default The content to be protected by the block mask.
[blockTemplate] Content projected into the mask overlay (e.g. a spinner or status message). Announced by screen readers via the mask's live region.

ARIA Attributes

Element Attribute When
Host (ui-lib-block-ui) aria-busy="true" blocked = true
Host aria-busy="false" blocked = false
Host aria-disabled="true" blocked = true (removed when false)
Mask element role="status" Always present
Mask element aria-live="polite" Always present — announces projected mask content to screen readers
Mask element aria-hidden="true" blocked = false (removed when blocked)
Content wrapper inert="" blocked = true — makes all descendants non-focusable

Keyboard Interaction

Interaction Behaviour
Tab / Shift+Tab Keyboard focus cannot enter the blocked content while blocked = true — the content wrapper receives inert, making all descendants non-focusable.
Any key No special handling on the mask overlay itself. All standard keyboard interactions remain available outside the blocked area.

Notes

  • Focus prevention: The protected content wrapper receives the inert HTML attribute when blocked = true. This prevents all interactive descendants (buttons, links, inputs) from receiving keyboard focus or pointer events — no custom focus trap is needed.
  • Live region: The mask element carries role="status" and aria-live="polite". Screen readers will announce any text projected via [blockTemplate] when the component becomes blocked (e.g. "Processing…").
  • aria-busy: Signals to assistive technologies that the container is loading or processing, suppressing interaction expectations.
  • All transitions respect prefers-reduced-motion: reduce — the opacity transition is set to 0 duration.
