BlockUI

Overlay a mask over any section to prevent user interaction — useful for loading states, disabled form sections, and confirmation flows. Wrap content in <ui-lib-block-ui> and bind [blocked] to toggle the mask.

Misc Overlay

Basic Usage

Toggle blocked to overlay a semi-transparent mask on the wrapped content. The content underneath remains in the DOM but pointer events are disabled.

Panel Content

This content is blocked when the mask is active. You cannot interact with the button below while blocked.

 
<ui-lib-block-ui [(blocked)]="isBlocked">
  <!-- your content -->
</ui-lib-block-ui>

Custom Mask Content

Project content into the mask using the blockTemplate attribute selector. This is ideal for spinners, progress indicators, or status messages.

Form Section

Submitting this form will block the section while processing.

Processing… 
<ui-lib-block-ui [blocked]="isBlocked">
  <!-- protected content -->
  <div blockTemplate>
    <!-- custom mask content, e.g. a spinner -->
  </div>
</ui-lib-block-ui>

API Reference

Input Type Default Description
blocked boolean false Whether the content is blocked. Supports two-way binding via [(blocked)].
variant 'material' | 'bootstrap' | 'minimal' | null null Visual variant. Inherits from ThemeConfigService when not set.
styleClass string | null null Additional CSS classes applied to the host element.
baseZIndex number 0 Base z-index for the mask layer. Uses the CSS variable default when 0.
Content Slot Description
default The content to be protected by the block mask.
blockTemplate Content projected into the mask overlay (e.g. a spinner or message).
