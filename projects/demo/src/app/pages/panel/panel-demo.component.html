Panel

import { Panel } from 'ui-lib-custom/panel'

A flexible content container with an optional collapsible body, a custom header slot, header action icons, and a footer slot. Supports three design variants and a smooth CSS grid-row collapse animation.

Basic

Set the header input to display a simple labelled panel. The body accepts any projected content.

The Panel component is a lightweight wrapper that groups related content under a labelled header. It renders as a bordered card with a header area, body, and optional footer.

Toggleable

Set [toggleable]="true" to display a chevron button in the header. Use [(collapsed)] for two-way binding and (toggled) to receive a PanelToggleEvent on every state change.

This content is collapsible. Click the chevron button in the header to toggle visibility. The collapse animation uses the CSS grid-template-rows technique for a smooth height transition.

The two-way [(collapsed)] binding keeps the parent in sync, and the (toggled) output emits a PanelToggleEvent on every change.

API

Inputs

InputTypeDefaultDescription
header string '' Header title text. Use [panelHeader] projection for rich HTML.
toggleable boolean false Whether the body can be collapsed and expanded.
collapsed boolean (model) false Two-way bound collapsed state. Use [(collapsed)] with a signal.
variant 'material' | 'bootstrap' | 'minimal' | null null Design variant. Falls back to ThemeConfigService when null.
styleClass string | null null Additional CSS classes added to the host element.

Outputs

OutputTypeDescription
toggled PanelToggleEvent Emitted when the collapsed state changes via the toggle button.

PanelToggleEvent shape: { collapsed: boolean; originalEvent?: Event }

Content Projection

SlotSelectorDescription
Body (default) Main body content.
Custom header [panelHeader] Replaces the text header area. The toggle button is preserved.
Header icons [panelIcons] Action elements placed in the header right area, before the toggle button.
Footer [panelFooter] Content below the body. Always visible regardless of collapsed state.

Notes

  • The panel host uses role="region" and aria-labelledby so screen readers can announce it as a named landmark. Only use Panel for content that warrants landmark navigation — avoid wrapping every minor section.
  • When collapsed, the content wrapper gains aria-hidden="true" so assistive technologies skip hidden body content entirely.
  • The toggle button uses aria-expanded to communicate current state; the value updates reactively as the panel opens and closes.
  • Focus is never moved programmatically on toggle — the user retains focus on the toggle button, which is the expected pattern for show/hide regions.
  • All collapse/expand animations respect @media (prefers-reduced-motion: reduce) — the transition duration is set to 0ms for users who prefer reduced motion.
