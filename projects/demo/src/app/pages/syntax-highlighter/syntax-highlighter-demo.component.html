SyntaxHighlighter

A lightweight, zero-dependency syntax tokenizer for TypeScript, HTML/Angular templates, and SCSS/CSS. No external libraries — pure regex tokenization with VS Code–inspired colours. Fully integrated with ui-lib-code-snippet.

Overview

The built-in highlighter powers the ui-lib-code-snippet component for TypeScript, HTML, SCSS, and CSS. You can also call it directly to highlight code in your own components or utilities — no Angular dependency, just a pure function.

API

highlight(code: string, language: SyntaxLanguage): string

Returns an HTML string with <span class="uilib-token-{type}"> wrappers. All code content is HTML-escaped before wrapping — the result is safe to set as innerHTML.

tokenize(code: string, language: SyntaxLanguage): readonly SyntaxToken[]

Returns the raw token array without converting to HTML. Each token has a type (e.g. 'keyword', 'string') and a value string. Useful when you need custom rendering.

escapeForCode(code: string): string

HTML-escapes a plain code string without any token spans. Use this for languages the built-in tokenizer does not support (JSON, Bash, shell, etc.).

SyntaxLanguage

type SyntaxLanguage = 'typescript' | 'scss' | 'css' | 'html'

Theming

Override any token colour via CSS custom properties on the host element or a parent container. All token colours are exposed as --uilib-syntax-* variables.
