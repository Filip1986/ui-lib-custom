Compact inline element representing an attribute, tag, or dismissible item. Supports an optional leading icon or image, a removable close button, three sizes, and three design variants.

Basic

Set the label input to display a simple text chip.

With Icon

Add an icon input (a PrimeIcons CSS class) to decorate the chip with a leading icon. The icon is decorative and hidden from assistive technology.

With Image

Provide an image URL to render a circular avatar thumbnail. Always supply a meaningful imageAlt description — the default "Chip" is not appropriate for real user images.

Removable

Set [removable]="true" to display a close button. Clicking it emits the (removed) event — the chip does not disappear automatically. Manage visibility in the parent component.

Selectable

Set [selectable]="true" to make the chip keyboard-focusable and toggleable. Bind [selected] and (selectedChange) to track state. Wrap selectable chips in a role="listbox" container so screen readers announce the collection correctly.

Three sizes: sm, md (default), and lg.

Design Variants

Three design variants: material (elevated, pill-shaped), bootstrap (badge-like, square corners), and minimal (muted surface with border). When variant is null (the default), it inherits from ThemeConfigService.

Inputs

Input Type Default Description
label string | null null Text displayed inside the chip.
icon string | null null CSS class for a leading icon (e.g. "pi pi-user"). Ignored when image is also set.
image string | null null URL of a circular thumbnail image rendered at the start of the chip.
imageAlt string "Chip" Alt text for the chip image. Provide a meaningful description for real user images.
removable boolean false When true, a close button is rendered. Emits (removed) on click.
removeIcon string "pi pi-times" CSS class for the remove button icon.
selectable boolean false When true, the chip is keyboard-focusable and toggleable. Pair with [selected] and (selectedChange).
selected boolean false Selected state for selectable chips.
size "sm" | "md" | "lg" "md" Size of the chip.
variant "material" | "bootstrap" | "minimal" | null null Design variant. Inherits from ThemeConfigService when null.
styleClass string | null null Additional CSS classes applied to the host element.

Outputs

Output Type Description
(removed) OutputEmitterRef<MouseEvent> Emitted when the remove button is clicked. The chip does not auto-hide — manage visibility in the parent.
(selectedChange) OutputEmitterRef<boolean> Emitted when a selectable chip is toggled; provides the new selected value.

CSS Custom Properties

ARIA Attributes

Attribute Element Value Notes
role Host option or group option for non-removable chips; group for removable chips to allow a nested interactive button.
aria-label Host Value of label input Omitted when label is null.

tabindex Host "0" Only present when [selectable]="true".

Keyboard Interaction

Notes

  • Wrap selectable chips in a container with role="listbox" and aria-label so screen readers announce the collection correctly.
  • The label input provides the accessible name via aria-label on the host element. When using the default content projection slot instead, set aria-label manually on the chip or its parent container.
  • Always provide a meaningful imageAlt for avatar images. The default "Chip" is not appropriate for real user photographs.
  • All transitions respect prefers-reduced-motion: reduce.
