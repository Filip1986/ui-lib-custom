FocusTrap

A directive that constrains keyboard focus within a container element. Tab and Shift+Tab cycle only among the focusable descendants of the host, preventing focus from escaping — essential for accessible modals, dialogs, and side panels.

Directive Misc A11y

Basic Usage

Add uiLibFocusTrap to any container. Tab navigation is constrained to focusable descendants of that element. Focus moves to the first focusable child on activation.

Toggle Trap

Bind [uiLibFocusTrap] to a boolean signal to enable or disable the trap at runtime. When disabled, Tab behaves normally and focus is restored to the previously focused element.

Modal Overlay Pattern

The most common use case: wrapping a modal dialog so that keyboard users cannot interact with background content. When the modal closes, focus returns to the trigger button automatically.

API

Import

Selector

Inputs

Name Type Default Description
uiLibFocusTrap boolean true When true, keyboard focus is trapped within the host element. Set to false to release the trap at runtime.

Keyboard behavior

Key Behavior
Tab Move focus to the next focusable descendant; wraps to first from last.
Shift + Tab Move focus to the previous focusable descendant; wraps to last from first.

Accessible elements included

The trap cycles through: a[href], button, input, select, textarea, elements with a non-negative tabindex, and [contenteditable="true"]. Disabled and aria-hidden elements are excluded.
