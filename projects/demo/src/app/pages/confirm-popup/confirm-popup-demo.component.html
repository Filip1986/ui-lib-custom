ConfirmPopup

A lightweight anchored popup for inline confirmations — positions itself next to the trigger element with an arrow indicator. No modal backdrop, less visual disruption than ConfirmDialog.

Basic

Call ConfirmPopupService.confirm({ target, message }) from a click handler, passing event.currentTarget as the anchor.

Confirm action

With Icon & Custom Labels

Pass an icon CSS class (PrimeIcons / FontAwesome) to render an icon beside the message. Customise button labels and severity independently.

Button Severities

The accept and reject buttons support six severity values: primary, secondary, success, danger, warning, info.

@for (severity of severities; track severity) { {{ severity | titlecase }} }

Design Variants

The popup supports three design variants: material (pill buttons, large radius), bootstrap (border, square corners), and minimal (subtle, flat). Each trigger below opens a popup in its respective variant.

@for (variant of variants; track variant) { {{ variant | titlecase }} }

Declarative Usage

Use [(visible)] for full two-way binding without the service. Outputs (accepted) and (rejected) let you react to user choices.
