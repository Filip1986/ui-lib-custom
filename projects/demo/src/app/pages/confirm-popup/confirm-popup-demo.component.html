The accept and reject buttons support six severity values: primary, secondary, success, danger, warning, info.

@for (severity of severities; track severity) { {{ severity | titlecase }} }

The popup supports three design variants: material (pill buttons, large radius), bootstrap (border, square corners), and minimal (subtle, flat). Each trigger below opens a popup in its respective variant.

@for (variant of variants; track variant) { {{ variant | titlecase }} }

Use [(visible)] for full two-way binding without the service. Outputs (accepted) and (rejected) let you react to user choices.

Inputs

Input Type Default Description
visible model<boolean> false Two-way visibility binding
key string '' Key for targeting a specific popup instance
message string 'Are you sure…' Confirmation message text
icon string | null null Icon CSS class before the message
acceptLabel string 'Yes' Accept button label
rejectLabel string 'No' Reject button label
acceptSeverity ConfirmPopupButtonSeverity 'primary' Accept button colour severity
rejectSeverity ConfirmPopupButtonSeverity 'secondary' Reject button colour severity
defaultFocus ConfirmPopupDefaultFocus 'accept' Button focused on open
variant ConfirmPopupVariant | null null Design variant (inherits from theme when null)
styleClass string | null null Extra CSS classes on host

Outputs

Output Payload Description
accepted void Emitted when accept button is clicked
rejected void Emitted when reject / overlay / Escape is triggered
