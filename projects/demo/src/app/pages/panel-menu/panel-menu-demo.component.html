An accordion-style, hierarchical navigation menu driven by a data model. Root items act as collapsible panels; leaf items are directly activatable. Implements the accordion + menu accessibility pattern.

Basic

Panel headers expand on click to reveal their child items.

Initially Expanded

Set expanded: true on a root item to have it open on first render.

Multiple Mode

By default only one panel can be open at a time. Set [multiple]="true" to allow multiple expanded panels simultaneously.

Nested Sub-Items

Items within a panel can also have their own items array, creating deeply nested collapsible groups.

Disabled Items

Set disabled: true on any item — root panels or leaf items — to prevent interaction. Disabled items receive aria-disabled="true".

URL Items

Provide url on a leaf item to render it as an anchor tag. Use target for link targeting.

Variants

Three design variants: material, bootstrap, and minimal. Falls back to the global ThemeConfigService variant when not set.

Inputs

NameTypeDefaultDescription
model PanelMenuItem[] [] Array of root-level menu items.
multiple boolean false Allow multiple root panels open at once.
variant 'material' | 'bootstrap' | 'minimal' | null null Design variant. Falls back to ThemeConfigService when null.
size 'sm' | 'md' | 'lg' 'md' Size token controlling font size and padding.

Outputs

NamePayloadDescription

Notes

  • Every component instance generates unique IDs for root header/content pairs so aria-controls ↔ id and aria-labelledby ↔ header id are always correctly wired.
  • Root header buttons keep toggle behaviour semantic and screen-reader friendly — they are never plain <div> elements.
  • Motion transitions are reduced or disabled under prefers-reduced-motion: reduce.
