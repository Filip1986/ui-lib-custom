Accessibility
  • Native semantics: a hidden native input owns checkbox ARIA semantics and keyboard behavior.
  • Keyboard: focus and toggle are driven by the input, with visible focus styling on the custom box.
  • Labeling: support inline label text, projected content, ariaLabel, and external label[for] through inputId.
  • Modes: binary mode uses scalar values, while group mode uses arrays; both are form-compatible.
